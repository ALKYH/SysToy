# SysToy 环境要求与支持边界

## 1. 目标

明确 SysToy 项目在 `Phase 1：环境搭建` 中的最小环境要求、支持边界和不支持范围。

## 2. 最小支持环境

本项目当前最小支持环境如下：

1. 操作系统：Windows 10
2. Linux 子系统：WSL2
3. 汇编工具：NASM
4. C 编译工具：GCC
5. 构建工具：Make
6. 模拟器：QEMU

## 3. 最小能力目标

学生环境只要满足以下能力，即视为达到 Phase 1 最低目标：

1. 能在本地执行构建命令。
2. 能成功调用 NASM、GCC、Make。
3. 能使用 QEMU 启动指定镜像。
4. 能提交环境自检结果。

## 4. 当前支持边界

### 4.1 支持

1. Windows 10 + WSL2 作为开发基础。
2. 使用命令行方式进行构建和运行。
3. 使用 QEMU 完成镜像加载与运行验证。

### 4.2 暂不要求

1. 不要求图形化 IDE 集成。
2. 不要求 Docker 化环境。
3. 不要求多平台兼容。
4. 不要求云端开发环境。

### 4.3 暂不支持为正式范围

1. 纯 Windows 原生命令行替代 WSL2 的完整兼容方案。
2. macOS 或 Linux 平台的课程标准支持。
3. 非 QEMU 的模拟器作为统一验收环境。

## 5. 环境交付要求

学生团队在环境阶段应提交：

1. 工具版本信息
2. 自检结果
3. 基线代码运行结果

## 6. 风险提醒

1. 环境变量配置错误会导致工具不可用。
2. QEMU 未正确安装会导致无法统一验收。
3. WSL2 未启用或异常会阻塞全部后续阶段。

## 7. 完成定义

当学生团队可在统一环境中完成构建、启动和结果展示时，视为 Phase 1 环境搭建完成。
