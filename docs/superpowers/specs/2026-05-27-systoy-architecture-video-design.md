# SysToy 项目架构讲解视频设计文档

## 1. 目标

为 `SysToy` 生成一支纯视觉讲解视频，面向已经具备基础系统知识的技术受众，用动画、字幕和架构图解释项目的整体架构与核心原理，不使用配音。

视频目标：

- 在 5 分钟左右建立对 SysToy 的整体认知
- 清晰解释启动、陷阱处理、系统调用、调度、同步、FAT32 与 ELF 加载等关键机制
- 体现该项目“教学操作系统”“阶段式演进”“可验证工程链路”三个特征

## 2. 输入依据

本视频以仓库内已有材料为基础进行重组与可视化，主要依据包括：

- `docs/prd-system-lab-unified-project.md`
- `docs/repository-structure-and-conventions.md`
- `projects/systoy-architecture_ppt169_20260527/design_spec.md`
- `kernel/`、`user/`、`tools/`、`tests/` 目录体现出的实际模块结构

## 3. 视频定位

- 形式：纯视觉架构解说版
- 时长：约 5 分钟
- 语言：中文字幕，无旁白
- 叙事方式：结构化蓝图版，结尾补充阶段演进时间线
- 观感目标：技术、清晰、克制、连续，不做花哨宣传片风格

## 4. 受众

主要受众：

- 操作系统课程教师
- 项目答辩评审
- 有基础系统背景的学生
- 需要快速理解项目架构的协作者

受众预期：

- 不要求了解全部实现细节
- 需要在较短时间内理解系统边界、模块关系和关键执行路径

## 5. 叙事主线

视频采用“从全局到局部，再回到全局”的结构：

1. 项目定位：说明 SysToy 是什么
2. 整体分层架构：先建立全景地图
3. 启动原理：说明系统如何从入口进入用户态
4. 系统调用与 Trap：说明用户态如何进入内核并返回
5. 调度与同步：说明多任务与共享资源控制
6. FAT32 与 ELF 加载：说明程序如何从文件系统进入内存并运行
7. 构建链路与阶段演进：说明它为何是一个可演进、可验证的教学工程

## 6. 视觉身份

### 6.1 风格

- 视觉方向：深色技术蓝图
- 关键词：结构感、工程感、秩序感、可解释性
- 禁止事项：炫技转场、过度粒子特效、复杂 3D 镜头、长段文字堆叠

### 6.2 色彩

- 背景色：`#0D1117`
- 次级背景：`#161B22`
- 主色：`#58A6FF`
- 强调色：`#3FB950`
- 事件警示色：`#F78166`
- 正文色：`#C9D1D9`
- 次级文字：`#8B949E`
- 边框与分隔线：`#30363D`

### 6.3 字体

- 中文主字体：`Microsoft YaHei`, `PingFang SC`, `sans-serif`
- 英文主字体：`Arial`, `sans-serif`
- 代码与寄存器名：`Consolas`, `Courier New`, `monospace`

### 6.4 字幕规则

- 每个镜头仅保留 1 个主标题
- 每个镜头最多 1 到 3 条短句字幕
- 单条字幕尽量控制在 8 到 18 个汉字的理解粒度
- 代码术语如 `ecall`、`mret`、`mepc`、`U-mode`、`M-mode` 使用等宽或高辨识写法

## 7. 动画语言

- 模块出现：上浮淡入、描边点亮、局部发光
- 数据流与控制流：沿路径脉冲前进
- 当前讲解对象：高亮边框、连线发光、局部放大
- 场景切换：擦除、推入、连线延展、层级折叠
- 节奏控制：关键图出现后保留 1 到 1.5 秒阅读停顿

动画原则：

- 动画服务于解释，不服务于炫耀
- 同一场景避免多个元素同时竞争注意力
- 所有切换都必须连续，不出现生硬跳切

## 8. 分镜设计

### Scene 1：封面开场

- 时长：8 秒
- 核心内容：`SysToy` 标题、副标题“教学操作系统的架构与原理”
- 画面形式：暗场中逐步显现项目名、网格与系统连线
- 作用：建立主题与技术气质

### Scene 2：项目定位

- 时长：18 秒
- 核心内容：用一句定义说明 SysToy 的定位
- 辅助标签：`RISC-V 32-bit`、`QEMU virt`、`C + 汇编`、`阶段式演进`
- 作用：让观众快速知道项目是什么

### Scene 3：整体分层架构

- 时长：30 秒
- 核心内容：四层结构逐层点亮
- 层级建议：
  - 硬件 / QEMU 层
  - 内核支撑层
  - Trap / 系统调用层
  - 用户程序层
- 作用：建立系统全貌

### Scene 4：启动链路

- 时长：35 秒
- 核心流程：
  - `_start`
  - `mtvec` / `mscratch` 初始化
  - `kmain()`
  - 加载 Task A / Task B
  - `enter_user_mode()`
  - `mret`
- 作用：解释系统如何从入口进入用户态执行

### Scene 5：Trap 与系统调用

- 时长：40 秒
- 核心流程：
  - 用户程序发出 `ecall`
  - `trap_entry` 保存上下文
  - `trap_dispatch`
  - `syscall_dispatcher`
  - `mret` 返回用户态
- 作用：解释用户态与内核态交互路径

### Scene 6：调度机制

- 时长：35 秒
- 核心流程：
  - Task A 运行
  - timer interrupt
  - 保存上下文
  - 切换 `current_task`
  - Task B 运行
- 作用：解释时间片轮转调度

### Scene 7：同步与共享内存

- 时长：28 秒
- 核心对比：
  - 无保护访问导致竞态
  - `sem_wait` / `sem_signal` 保护下的安全访问
- 视觉策略：红色展示冲突，绿色展示正确同步
- 作用：解释为什么需要信号量与共享资源控制

### Scene 8：FAT32 与 ELF 加载

- 时长：42 秒
- 核心流程：
  - 虚拟磁盘布局
  - 目录项匹配
  - ELF 校验
  - 段加载进入内存
  - 用户程序可执行
- 作用：解释程序如何从文件系统进入执行态

### Scene 9：构建流水线

- 时长：28 秒
- 核心流程：
  - 汇编 / 编译
  - 用户 ELF
  - FAT32 镜像
  - 二进制嵌入
  - 链接
  - QEMU 运行
- 作用：体现工程化构建链路

### Scene 10：阶段演进收束

- 时长：22 秒
- 核心内容：
  - Phase 0 到 Phase 6 时间线
  - 环境、启动、内存、调度、同步、文件系统逐步点亮
- 作用：体现项目的课程化演进特征

### Scene 11：结尾总结

- 时长：10 秒
- 核心内容：用一句话总结项目定位与能力边界
- 作用：收束并回到全局架构

## 9. 画面内容约束

- 优先使用架构图、流程图、模块卡片、时间线和流水线图
- 不使用大段代码滚屏
- 可少量引入关键术语和简化结构片段，但不做长代码讲解
- 所有中文内容统一使用 UTF-8 编码
- 中文文案优先保证可读性和术语稳定性

## 10. HyperFrames 实现策略

### 10.1 工程组织

建议在仓库内新建独立 HyperFrames 项目目录，用于承载：

- `index.html`
- `compositions/`
- `DESIGN.md`
- 渲染输出目录

### 10.2 结构建议

- Root composition 负责总时间轴编排
- 各核心段落拆为独立 composition
- 多场景结构必须使用明确转场，禁止跳切

### 10.3 资产策略

- 以 SVG 原生图形、文本、连线、卡片和简单装饰为主
- 尽量不依赖外部图片
- 若使用现有设计稿元素，应优先转为可控的 HTML/SVG 表达

## 11. 交付物

本任务的目标交付物包括：

- 一套 HyperFrames 视频工程
- 可预览的本地项目
- 最终导出的视频文件
- 如有需要，可附简短说明文档记录运行与渲染方式

## 12. 验收标准

视频应满足以下标准：

- 能在无旁白条件下独立传达项目架构与核心原理
- 结构完整，镜头之间叙事连续
- 文案简洁，可在镜头停留时间内读完
- 架构图、流程图和关键路径表达清晰
- 风格统一，无编码乱码，无中文显示异常
- 能通过 HyperFrames 的基本 lint、validate、inspect 流程

## 13. 风险与应对

### 风险 1：纯视觉信息过载

应对：

- 严控单镜头信息量
- 保持“一个镜头只回答一个问题”

### 风险 2：无配音导致理解断层

应对：

- 使用短句字幕
- 增加状态角标与路径高亮
- 在关键节点留出阅读停顿

### 风险 3：中文排版或编码异常

应对：

- 全流程统一 UTF-8
- 预览与渲染阶段重点检查中文文本显示

## 14. 实施边界

本次视频聚焦“架构和原理讲解”，不包含：

- 课程宣传片式包装
- 完整代码教程
- 录屏式终端演示
- 配音生成与字幕对齐流程

## 15. 结论

本视频将采用“结构化蓝图版”作为主叙事方式，以深色技术蓝图为统一视觉身份，在约 5 分钟内完成对 SysToy 架构与原理的分层解释，并通过 HyperFrames 生成纯视觉讲解视频。
