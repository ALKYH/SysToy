# SysToy 基线运行验收标准

## 1. 目标

定义 Phase 1 中“环境已经跑通”的统一判定标准。

## 2. 最低通过标准

一个团队若满足以下全部条件，可判定为通过基线运行验收：

1. 工具链检查通过。
2. 基线代码能够成功构建。
3. 能生成目标镜像或等价产物。
4. QEMU 能加载镜像并正常启动。
5. 运行输出中能看到预期关键结果。

## 3. 编译通过标准

满足以下条件可视为编译通过：

1. 构建过程完成。
2. 没有阻塞性错误。
3. 目标产物成功生成。

## 4. 启动通过标准

满足以下条件可视为启动通过：

1. QEMU 成功打开目标镜像。
2. 系统没有立即退出或崩溃。
3. 能看到预期启动输出。

## 5. 结果证明材料

建议至少提交以下材料中的两项：

1. 工具版本截图或文本记录
2. 构建成功截图或日志
3. QEMU 启动截图或录像
4. 自检清单

## 6. 不通过判定

出现以下情况之一可判定为不通过：

1. 工具链缺失
2. 无法构建
3. 无法生成镜像
4. QEMU 无法加载
5. 无法展示关键输出

## 7. 助教验收建议

助教验收时优先看：

1. 是否真的能现场运行
2. 是否能复述最小环境结构
3. 是否提交完整证明材料
