# SysToy Phase 0-1 交付说明

## 1. 交付范围

本次交付覆盖 `Phase 0：项目基线与治理准备` 和 `Phase 1：环境搭建`。

## 2. 已交付内容

### 2.1 Phase 0：项目基线与治理准备

1. 仓库结构与命名规范：
   - `docs/repository-structure-and-conventions.md`
2. 阶段验收模板：
   - `docs/templates/stage-acceptance-template.md`
   - `docs/templates/stage-demo-record-template.md`
   - `docs/templates/issue-log-template.md`
3. 评分 Rubric：
   - `docs/scoring-rubric.md`

### 2.2 Phase 1：环境搭建

1. 环境目标与支持边界说明：
   - `docs/environment-requirements.md`
2. 环境自检清单：
   - `docs/environment-self-checklist.md`
3. 基线运行验收标准：
   - `docs/baseline-run-acceptance.md`

## 3. 使用建议

1. 课程负责人先阅读仓库规范和评分 Rubric。
2. 助教在课程开始前复核环境要求与验收口径。
3. 学生团队先完成环境自检，再按基线运行验收标准提交结果。
4. 每阶段统一使用模板记录验收、演示和问题。

## 4. 完成定义

当以下条件满足时，视为 Phase 0-1 已具备执行条件：

1. 团队知道仓库目录、命名、编码和交付规则。
2. 教师和助教能够按统一模板验收和记录。
3. 学生能够明确环境边界与最小工具要求。
4. 学生能够依据自检清单确认环境状态。
5. 学生能够依据基线运行标准判断“环境是否跑通”。
